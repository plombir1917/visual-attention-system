# ФОКУС — система определения внимания

ФОКУС определяет, сосредоточен ли человек на экране, по изображению с обычной
веб-камеры. Система анализирует направление взгляда и положение головы, считает
угол отклонения от экрана и на его основе принимает решение: внимание есть или
человек отвлёкся. Никакого специального оборудования не требуется — достаточно
камеры ноутбука.

VAS (Visual Attention Sistem) - система визуального внимания, рабочее название.

Кадры обрабатываются на лету и не сохраняются: на сервере остаются только
числовые метрики внимания, привязанные к сессии и учётной записи.

## Как это устроено

Система состоит из нескольких независимых сервисов, общающихся между собой по
gRPC, WebSocket и через Kafka:

```
Клиент (виджет в браузере или десктоп-приложение)
   │  кадр JPEG, ~1 кадр/с по WebSocket
   ▼
attention-service (Go, :8080)         relay + авторизация по API-ключу
   │  gRPC-стрим
   ▼
attention-model (Python, :50051)      ML-пайплайн
   │     ├─ MediaPipe       — сетка лица, ориентиры глаз
   │     ├─ GazeCNN (ResNet)— вектор взгляда
   │     ├─ solvePnP        — поза головы
   │     └─ геометрия конуса— решение «внимание / нет»
   ▼
   метрики возвращаются клиенту, кешируются в Redis,
   а по завершении сессии публикуются в Kafka
   ▼
analytics-service (Go)                консьюмер Kafka → PostgreSQL
```

Веб-сайт, личный кабинет и админ-панель живут в `user-service`; он же отвечает
за регистрацию пользователей и выдачу API-ключей.

## Сервисы

| Сервис              | Стек                       | Порт         | Назначение                                  |
| ------------------- | -------------------------- | ------------ | ------------------------------------------- |
| `attention-model`   | Python, FastAPI, PyTorch   | 8765 / 50051 | ML-пайплайн: кадр → метрики внимания        |
| `attention-service` | Go                         | 8080         | Приём кадров, авторизация, кеш, события     |
| `analytics-service` | Go                         | —            | Складывает метрики сессий в PostgreSQL      |
| `user-service`      | TypeScript, NestJS, Prisma | 3000         | Сайт, личный кабинет, админка, пользователи |

Инфраструктура: PostgreSQL, Redis, Kafka (и Kafka UI на `:8090`).

## Клиенты

В каталоге `clients/` лежат два способа подключиться к системе:

- `widget` — npm-пакет `vas-widget`. Встраиваемый виджет для любого фронтенда
  (React, Vue, Svelte, чистый JS). Подключается одной строкой, показывает живой
  индикатор внимания.
- `desktop` — десктопное приложение на Electron. Работает без браузера,
  анализирует камеру в фоне.
